## 🔧 Common Issues & Fixes

### ❌ "Disk Drill cannot access this drive"

**Cause:** macOS privacy restrictions preventing full disk access  
**Fix:** Go to **System Settings** → **Privacy & Security** → **Full Disk Access**. Ensure Disk Drill is listed and toggled ON. If not listed, click the + button and add it from Applications.

---

### ❌ Scan runs but finds no files

**Cause:** The drive may be physically damaged, or data has been overwritten  
**Fix:** First, try Deep Scan instead of Quick Scan. If that fails, use the Byte-to-Byte Backup tool to create a disk image, then scan the image instead of the original drive. For SSDs with TRIM enabled, data recovery success rates are naturally lower because TRIM actively erases deleted data.

---

### ❌ Disk Drill crashes or shows spinning rainbow wheel

**Cause:** Interrupted scan process or resource contention  
**Fix:** Force quit Disk Drill (Option+Command+Escape), relaunch, and resume your scan session. The software saves scan progress, so you can pick up where you left off. If crashes persist, try scanning a byte-to-byte backup image instead of the original drive.

---

### ❌ macOS Installer utility fails to create bootable drive

**Cause:** Known limitation in current versions  
**Fix:** This is a documented issue — the macOS Installer utility within Disk Drill has been reported to fail at creating bootable volumes. Use Apple's official `createinstallmedia` command in Terminal or third-party tools like Etcher for bootable drive creation instead.

---

### ❌ "This disk is write-protected" when trying to recover

**Cause:** The drive has hardware-level write protection or is failing  
**Fix:** For SD cards, check the physical lock switch. For other drives, use Disk Drill's Byte-to-Byte Backup to create an image, then recover from the image file. This bypasses write-protection issues entirely.

### Q2: Can Disk Drill recover files after I've emptied the Trash?

Yes — emptying the Trash doesn't actually erase file data from your drive. It simply marks the space as available for new data. As long as those sectors haven't been overwritten, Disk Drill can recover the files. For best results, stop using the drive immediately after deletion — every new file you save risks overwriting the deleted data you want back.

### Q3: What's the difference between Quick Scan and Deep Scan?

- **Quick Scan** reads the file system's existing directory structures. It's fast and recovers recently deleted files with their original names intact. Best for "I just deleted something 10 minutes ago."
- **Deep Scan** ignores the file system entirely and reads every sector of the drive, looking for file signatures (unique patterns at the beginning of JPEGs, PDFs, etc.). It takes hours instead of minutes but can recover files from formatted drives, corrupted partitions, and older deletions.

Use Quick Scan first. If that doesn't find what you need, run Deep Scan.
